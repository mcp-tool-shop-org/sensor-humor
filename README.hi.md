एमसीपी टूल जो आपके एलएलएम को एक स्थायी हास्यपूर्ण साथी प्रदान करता है: मूड-आधारित व्यक्तित्व, सत्र-जागरूक कॉलबैक, लगातार चलने वाले चुटकुले, व्यंग्य, ताने और आकर्षक वाक्यांश - ये सभी पाईपर टीटीएस (लय-नियंत्रित) के माध्यम से आवाज एकीकरण के साथ।

डेवलपर्स के लिए बनाया गया: कोड में मौजूद कमियों पर हल्की प्रतिक्रिया, सरल और स्पष्ट त्रुटि संदेश, निर्माण विफल होने पर अराजक वृद्धि। यह कभी भी होस्ट एलएलएम की शैली को अधिलेखित नहीं करता - एक विशिष्ट आवाज जो कॉल किए जाने पर सुनाई देती है।

## विशेषताएं

- 6 मूड, प्रत्येक को अनुमानित और उच्च गुणवत्ता वाले आउटपुट के लिए रिक्त स्थान भरने वाले टेम्पलेट संकेत के साथ समायोजित किया गया है।
- सत्र स्थिति: लगातार चलने वाले चुटकुले, हालिया अंशों का बफर (अधिकतम 20), वाक्यांश मानचित्र - वैकल्पिक रूप से डिस्क पर संग्रहीत (`SENSOR_HUMOR_PERSIST`) ताकि कॉलबैक सर्वर रीस्टार्ट होने पर भी बने रहें।
- 9 उपकरण: मूड_सेट/मूड_गेट, हास्य समय, व्यंग्य, ताना, वाक्यांश_उत्पन्न/वाक्यांश_कॉलबैक, डिबग_स्थिति, सत्र_रीसेट
- स्थानीय ओलामा बैकएंड (qwen2.5:7b डिफ़ॉल्ट, `SENSOR_HUMOR_MODEL` के माध्यम से कॉन्फ़िगर करने योग्य)
- आवाज युग्मन: एमसीपी-वॉइस-साउंडबोर्ड पाईपर टीटीएस (लय नियंत्रण: लंबाई_स्केल, शोर_स्केल, शोर_डब्ल्यू_स्केल, वॉल्यूम) के साथ।
- नियतात्मक: जेएसओएन स्कीमा प्रवर्तन, सत्यापन, खराब आउटपुट पर पुनः प्रयास, मूड विरासत लागू।

## मूड

प्रत्येक मूड एक रिक्त स्थान भरने वाले टेम्पलेट संकेत का उपयोग करता है जो मॉडल को अनुमानित और उच्च गुणवत्ता वाली स्थिति में लाने के लिए मजबूर करता है।

- **सूखा** - सपाट, न्यूनतम, दर्दनाक रूप से स्पष्ट (डिफ़ॉल्ट)
- **व्यंग्य** - स्नेही, तीक्ष्ण व्यंग्य, निर्णय/निदान लेबल
- **संशयवादी** - निराश, शांत और क्रूर यथार्थवाद ("निश्चित रूप से:", "अनुमानित रूप से:")
- **शरारती** - चंचल, मजाकिया शरारत ("ओह हनी", "साहसी कदम")
- **अराजक** - एक ठोस वाक्य, फिर अचानक बेतुका मोड़ ("रिपोर्ट के अनुसार...")
- **ज़ूमर** - अत्यधिक ऑनलाइन, क्रूर पीढ़ी-जेड व्यंग्य (प्रतिक्रिया, ताना, कैप्स लॉक, टैग)

सभी मूड एमसीपी-वॉइस-साउंडबोर्ड (अनुशंसित पाईपर) के माध्यम से आवाज + लय को प्राप्त करते हैं।

## आवश्यकताएं

- नोड.जेएस 18+
- ओलामा स्थानीय रूप से `qwen2.5:7b` के साथ चल रहा है (या एक अलग मॉडल के लिए `SENSOR_HUMOR_MODEL` सेट करें)
- एमसीपी-वॉइस-साउंडबोर्ड स्थापित और चल रहा है (अनुशंसित पाईपर बैकएंड, वैकल्पिक)
- @modelcontextprotocol/sdk

## स्थापना

```bash
npm install @mcptoolshop/sensor-humor
# or install a local dev checkout
npm install /path/to/sensor-humor
```

### डॉकर

प्रत्येक रिलीज़ पर एक कंटेनर छवि जीएचसीआर में प्रकाशित की जाती है। सेंसर-ह्यूमर एसटीडीआईओ के माध्यम से एमसीपी बोलता है, इसलिए इसे इंटरैक्टिव रूप से चलाएं और इसे ओलामा तक पहुंचाएं:

```bash
docker run -i --rm -e OLLAMA_HOST=http://host.docker.internal:11434 \
  ghcr.io/mcp-tool-shop-org/sensor-humor:latest
```

## त्वरित शुरुआत

1. ओलामा शुरू करें:

```bash
ollama pull qwen2.5:7b
```

2. सेंसर-ह्यूमर एमसीपी सर्वर (एसटीडीआईओ परिवहन) शुरू करें:

```bash
cd sensor-humor
SENSOR_HUMOR_DEBUG=true npm start
```

3. वॉइस-साउंडबोर्ड (पाईपर मोड) शुरू करें:

```bash
cd ../mcp-voice-soundboard
VOICE_SOUNDBOARD_ENGINE=piper VOICE_SOUNDBOARD_PIPER_MODEL_DIR=/path/to/piper/models npm start
```

4. अपने एमसीपी क्लाइंट (क्लाउड कोड, कर्सर, आदि) में:
- दोनों सर्वर जोड़ें
- परीक्षण श्रृंखला:

```
mood_set(style: "roast")
roast(target: "800-line god function")
```

पाठ व्यंग्य लौटाया गया। यदि [mcp-voice-soundboard](https://github.com/mcp-tool-shop-org/mcp-voice-soundboard) भी कॉन्फ़िगर किया गया है, तो `voice_speak(mood: "roast")` इसे मूड के अनुसार उपयुक्त पाईपर लय के साथ बोलेगा।

## उपकरण

सभी उपकरण सत्र से वर्तमान मूड प्राप्त करते हैं।

| उपकरण | हस्ताक्षर | विवरण |
|------|-----------|-------------|
| `mood_set` | `(style: string)` | सक्रिय मूड सेट करें (सूखा, व्यंग्य, अराजक, शरारती, संशयवादी, ज़ूमर) |
| `mood_get` | `()` | वर्तमान मूड + चुटकुला गणना |
| `comic_timing` | `(text, technique?)` | हास्यपूर्ण तरीके से फिर से लिखें (तीन का नियम, दिशाभ्रम, वृद्धि, कॉलबैक, कम करके आंकना, ऑटो) |
| `roast` | `(target, context?)` | वर्तमान मूड की आवाज में स्नेही व्यंग्य, 1-5 की गंभीरता लौटाता है। संदर्भ: कोड, त्रुटि, विचार, स्थिति |
| `heckle` | `(target)` | संक्षिप्त, तीक्ष्ण ताना |
| `catchphrase_generate` | `(context?)` | पुन: प्रयोज्य अंश बनाएं (सत्र में संग्रहीत) |
| `catchphrase_callback` | `()` | सबसे अधिक उपयोग किए जाने वाले वाक्यांश का पुन: उपयोग करें (या शून्य) |
| `debug_status` | `()` | लाइव बैकएंड स्वास्थ्य (ओलामा पहुंच योग्य, मॉडल खींचा गया), हल किया गया कॉन्फ़िगरेशन, फ़ॉलबैक गणना और सत्र स्थिति |
| `session_reset` | `()` | सभी सत्र स्थितियों को रीसेट करें (मूड, चुटकुले, अंश, वाक्यांश, मोड़ काउंटर) |

**खराब आउटपुट:** जब ओलामा पहुंच योग्य नहीं होता है या मॉडल खींचा नहीं जाता है, तो हास्य उपकरण एक इन-वॉइस डिब्बाबंद पंक्ति और `degraded: true` और एक `degraded_reason` (`connection`, `model-not-found`, `timeout`, `safety-filter`, ...) लौटाते हैं ताकि कॉलर वास्तविक मजाक को फ़ॉलबैक से अलग कर सके - एक वास्तविक मॉडल पीढ़ी में कोई `degraded` ध्वज नहीं होता है। लाइव पहुंच क्षमता, हल किया गया मॉडल/होस्ट/टाइमआउट और फ़ॉलबैक गणना देखने के लिए `debug_status` कॉल करें। `roast` और `heckle` भी सक्रिय `mood` को प्रतिध्वनित करते हैं; `catchphrase_generate` `is_fresh` लौटाता है (`true` = नया बनाया गया, `false` = एक मौजूदा सत्र वाक्यांश का पुन: उपयोग किया गया)।

## मूड लय (पाईपर आवाज)

प्रत्येक मूड एक विशिष्ट पाईपर आवाज + लय कॉन्फ़िगरेशन पर मैप करता है:

| मूड | आवाज | लंबाई_स्केल | शोर_स्केल | शोर_डब्ल्यू_स्केल | वॉल्यूम | चरित्र |
|------|-------|-------------|-------------|---------------|--------|-----------|
| सूखा | en_GB-alan-medium | 1.15 | 0.3 | 0.3 | 0.9 | सपाट, थका हुआ, लयबद्ध |
| व्यंग्य | en_US-ryan-high | 0.95 | 0.667 | 0.8 | 1.0 | आत्मविश्वासी व्यंग्य |
| अराजक | en_US-lessac-high | 0.88 | 0.8 | 0.9 | 1.1 | समाचार एंकर निरर्थक बातें सुना रहा है |
| शरारती | en_GB-cori-high | 1.05 | 0.5 | 0.6 | 0.95 | गर्म, मजाकिया, चंचल इशारा |
| संशयवादी | en_GB-alan-medium | 1.25 | 0.2 | 0.2 | 0.8 | ठंडा, सपाट, शून्य आश्चर्य |
| ज़ूमर | en_US-lessac-high | 0.90 | 0.85 | 0.9 | 1.15 | तेज, जोर से, स्ट्रीमर ऊर्जा |

## पर्यावरण चर

```bash
# sensor-humor
SENSOR_HUMOR_DEBUG=true                # verbose prompt/response dumps
SENSOR_HUMOR_TIMEOUT_MS=30000          # Ollama call timeout in ms (default: 30000; invalid values fall back to default)
SENSOR_HUMOR_TEMPERATURE=0.55          # generation temperature, clamped 0.0-2.0 (default: 0.55)
SENSOR_HUMOR_PROMPT_VERSION=1          # prompt set version (only v1 ships today; other values fall back to v1)
SENSOR_HUMOR_MODEL=qwen2.5:7b         # Ollama model (default: qwen2.5:7b)
SENSOR_HUMOR_PERSIST=false             # persist session to ~/.sensor-humor/session.json (survives restart; 24h expiry)
SENSOR_HUMOR_SESSION_DIR=              # override the session directory (default: ~/.sensor-humor)
OLLAMA_HOST=http://127.0.0.1:11434    # Ollama API host (default: http://127.0.0.1:11434)
OLLAMA_API_KEY=                        # Bearer token for a remote/cloud Ollama (e.g. https://ollama.com); unset for local

# voice integration (in voice-soundboard)
VOICE_SOUNDBOARD_ENGINE=piper          # or kokoro (default)
VOICE_SOUNDBOARD_PIPER_MODEL_DIR=/path/to/piper/models
```

## अवलोकनीयता और डिबग

- प्रत्येक उपकरण कॉल संकेत भेजता है, कच्चा ओलामा प्रतिक्रिया, पार्स किया गया आउटपुट, सत्र अद्यतन लॉग करता है।
- आवाज: डिबग लॉग मूड के अनुसार लागू किए गए पाईपर पैरामीटर दिखाते हैं।
- `SENSOR_HUMOR_DEBUG=true` सेट करें ताकि सब कुछ दिखाई दे।

## गुणवत्ता नोट्स

- हास्य की गुणवत्ता कंकाल-आधारित प्रॉम्प्ट इंजीनियरिंग से आती है, न कि किसी एक मॉडल सेटिंग से — प्रत्येक मनोदशा एक अनुमानित आकार को मजबूर करती है। अपने स्वयं के मॉडल/हार्डवेयर पर हिट दर को `scripts/ab-scorecard.ts` (SCORECARD.md में टेम्पलेट) के साथ मापें।
- उपमा/तुलना फ़िल्टर: पोस्ट-वैलिडेशन रेगुलर एक्सप्रेशन + पुनः प्रयास, फिर यदि कोई रिसाव बना रहता है तो एक सुरक्षित फॉलबैक जो मनोदशा के अनुसार आवाज देता है।
- कठोर भाषा फ़िल्टर एक टर्मिनल गेट के रूप में चलता है, इसलिए अपशब्द उपयोगकर्ता तक नहीं पहुंच सकते हैं, भले ही देर से किए गए पुनः प्रयास में वे वापस आ जाएं।
- नियतात्मक: JSON स्कीमा प्रवर्तन, खराब आउटपुट पर पुनः प्रयास, सभी उपकरणों में मनोदशा विरासत को लागू किया जाता है।
- आवाज: पाइपर मनोदशा के अनुसार उच्चारण (लंबाई/शोर/वॉल्यूम) प्रदान करता है; कोकोरो फॉलबैक केवल गति पर आधारित है।
- केवल डेवलपर टूल साइडकिक। हास्य व्यक्तिपरक है; यदि आवश्यक हो तो किसी भी मनोदशा को पर्यावरण चर के माध्यम से अक्षम करें या प्रॉम्प्ट को समायोजित करें।

## सुरक्षा और विश्वास

- **डिफ़ॉल्ट रूप से स्थानीय** — HTTP के माध्यम से `localhost` पर ओलामा से बात करता है। `OLLAMA_HOST` किसी अन्य स्थान (जैसे, एक दूरस्थ/क्लाउड ओलामा) की ओर इशारा कर सकता है; यह एकमात्र बाहरी आउटपुट है और ऑपरेटर का स्पष्ट विकल्प है।
- **फ़ाइल सिस्टम** — डिफ़ॉल्ट रूप से कोई नहीं। `SENSOR_HUMOR_PERSIST=true` के साथ, यह एक फ़ाइल पढ़ता/लिखता है, `~/.sensor-humor/session.json` (डायरेक्टरी को `SENSOR_HUMOR_SESSION_DIR` के साथ ओवरराइड करें), जिसमें केवल आपके सत्र की हास्य स्थिति (बिट्स, चुटकुले, आकर्षक वाक्यांश) शामिल हैं — कोई क्रेडेंशियल नहीं। फ़ाइल 24 घंटे बाद स्वचालित रूप से समाप्त हो जाती है।
- **गुप्त जानकारी** — डिफ़ॉल्ट रूप से कोई नहीं। यदि आप `OLLAMA_HOST` को एक दूरस्थ/क्लाउड ओलामा की ओर इंगित करते हैं, तो `OLLAMA_API_KEY` सेट करें; इसे पर्यावरण से पढ़ा जाता है और केवल उस होस्ट पर `Bearer` हेडर के रूप में भेजा जाता है — कभी भी लॉग नहीं किया जाता, संग्रहीत नहीं किया जाता या प्रतिध्वनित नहीं किया जाता (`debug_status` केवल यह रिपोर्ट करता है कि कोई कुंजी सेट है या नहीं, इसका मान नहीं)।
- **कोई टेलीमेट्री नहीं** — कुछ भी एकत्र या भेजा नहीं जाता है।
- **सत्र स्थिति डिफ़ॉल्ट रूप से मेमोरी में होती है** — जब सर्वर प्रक्रिया बंद हो जाती है तो यह समाप्त हो जाती है; `SENSOR_HUMOR_PERSIST` के साथ डिस्क पर बने रहने का विकल्प चुनें।
- **इनपुट सैनिटाइजेशन** — प्रॉम्प्ट इंजेक्शन से पहले सभी उपयोगकर्ता द्वारा प्रदान किए गए टेक्स्ट को सैनिटाइज किया जाता है (नई लाइनें हटा दी जाती हैं, लंबाई सीमित की जाती है, नियंत्रण वर्ण हटा दिए जाते हैं)।
- **आउटपुट फ़िल्टरिंग** — कठोर भाषा फ़िल्टर (बेस64-एन्कोडेड टर्म सूची) के साथ पुनः प्रयास + एक टर्मिनल सुरक्षित फॉलबैक गेट अपशब्दों को उपयोगकर्ता तक पहुंचने से रोकता है, भले ही देर से किए गए पुनः प्रयास में या अपशब्द वाले इनपुट से।

## आर्किटेक्चर

```
Host LLM (Claude, etc.)
  | calls tool
  v
sensor-humor MCP server (TypeScript, stdio)
  | builds mood prompt + session state
  v
Ollama (qwen2.5:7b-instruct, local)
  | returns JSON (schema-enforced)
  v
sensor-humor validates -> updates session -> returns to host
  | host optionally calls
  v
mcp-voice-soundboard (Piper backend)
  | maps mood -> prosody preset
  v
Piper TTS (local ONNX) -> audio
```
