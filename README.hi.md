एमसीपी टूल जो आपके एलएलएम को एक स्थायी हास्यपूर्ण साथी प्रदान करता है: मूड-आधारित व्यक्तित्व, सत्र-जागरूक कॉलबैक, लगातार चलने वाले चुटकुले, व्यंग्य, ताने और आकर्षक वाक्यांश - ये सभी पाईपर टीटीएस (लय-नियंत्रित) के माध्यम से आवाज एकीकरण के साथ।

डेवलपर्स के लिए बनाया गया: कोड में मौजूद कमियों पर हल्की प्रतिक्रिया, सरल और स्पष्ट त्रुटि संदेश, निर्माण विफल होने पर अराजक वृद्धि। यह कभी भी होस्ट एलएलएम की शैली को अधिलेखित नहीं करता - एक विशिष्ट आवाज जो कॉल किए जाने पर सुनाई देती है।

## विशेषताएं

- 6 मूड, प्रत्येक को अनुमानित और उच्च गुणवत्ता वाले आउटपुट के लिए रिक्त स्थान भरने वाले टेम्पलेट संकेत के साथ समायोजित किया गया है।
- सत्र स्थिति: लगातार चलने वाले चुटकुले, हालिया अंशों का बफर (अधिकतम 20), वाक्यांश मानचित्र - वैकल्पिक रूप से डिस्क पर संग्रहीत (`SENSOR_HUMOR_PERSIST`) ताकि कॉलबैक सर्वर रीस्टार्ट होने पर भी बने रहें।
- 9 उपकरण: मूड_सेट/मूड_गेट, हास्य समय, व्यंग्य, ताना, वाक्यांश_उत्पन्न/वाक्यांश_कॉलबैक, डिबग_स्थिति, सत्र_रीसेट
- स्थानीय ओलामा बैकएंड (qwen2.5:7b डिफ़ॉल्ट, `SENSOR_HUMOR_MODEL` के माध्यम से कॉन्फ़िगर करने योग्य)
- आवाज युग्मन: एमसीपी-वॉइस-साउंडबोर्ड पाईपर टीटीएस (लय नियंत्रण: लंबाई_स्केल, शोर_स्केल, शोर_डब्ल्यू_स्केल, वॉल्यूम) के साथ।
- नियतात्मक: जेएसओएन स्कीमा प्रवर्तन, सत्यापन, खराब आउटपुट पर पुनः प्रयास, मूड विरासत लागू।

## मूड

प्रत्येक मूड एक रिक्त स्थान भरने वाले टेम्पलेट संकेत का उपयोग करता है जो मॉडल को अनुमानित और उच्च गुणवत्ता वाली स्थिति में लाने के लिए मजबूर करता है।

- **सूखा** - सपाट, न्यूनतम, दर्दनाक रूप से स्पष्ट (डिफ़ॉल्ट)
- **व्यंग्य** - स्नेही, तीक्ष्ण व्यंग्य, निर्णय/निदान लेबल
- **संशयवादी** - निराश, शांत और क्रूर यथार्थवाद ("निश्चित रूप से:", "अनुमानित रूप से:")
- **शरारती** - चंचल, मजाकिया शरारत ("ओह हनी", "साहसी कदम")
- **अराजक** - एक ठोस वाक्य, फिर अचानक बेतुका मोड़ ("रिपोर्ट के अनुसार...")
- **ज़ूमर** - अत्यधिक ऑनलाइन, क्रूर पीढ़ी-जेड व्यंग्य (प्रतिक्रिया, ताना, कैप्स लॉक, टैग)

सभी मूड एमसीपी-वॉइस-साउंडबोर्ड (अनुशंसित पाईपर) के माध्यम से आवाज + लय को प्राप्त करते हैं।

## आवश्यकताएं

- नोड.जेएस 18+
- ओलामा स्थानीय रूप से `qwen2.5:7b` के साथ चल रहा है (या एक अलग मॉडल के लिए `SENSOR_HUMOR_MODEL` सेट करें)
- एमसीपी-वॉइस-साउंडबोर्ड स्थापित और चल रहा है (अनुशंसित पाईपर बैकएंड, वैकल्पिक)
- @modelcontextprotocol/sdk

प्रत्येक रिलीज़ पर एक कंटेनर छवि जीएचसीआर में प्रकाशित की जाती है। सेंसर-ह्यूमर एसटीडीआईओ के माध्यम से एमसीपी बोलता है, इसलिए इसे इंटरैक्टिव रूप से चलाएं और इसे ओलामा तक पहुंचाएं:

```bash

  ghcr.io/mcp-tool-shop-org/sensor-humor:latest
```

## त्वरित शुरुआत

1. ओलामा शुरू करें:

```bash
ollama pull qwen2.5:7b
```

2. सेंसर-ह्यूमर एमसीपी सर्वर (एसटीडीआईओ परिवहन) शुरू करें:

```bash
cd sensor-humor
SENSOR_HUMOR_DEBUG=true npm start
```

3. वॉइस-साउंडबोर्ड (पाईपर मोड) शुरू करें:

```bash
cd ../mcp-voice-soundboard
VOICE_SOUNDBOARD_ENGINE=piper VOICE_SOUNDBOARD_PIPER_MODEL_DIR=/path/to/piper/models npm start
```

4. अपने एमसीपी क्लाइंट (क्लाउड कोड, कर्सर, आदि) में:
- दोनों सर्वर जोड़ें
- परीक्षण श्रृंखला:

```
mood_set(style: "roast")
roast(target: "800-line god function")
```

पाठ व्यंग्य लौटाया गया। यदि [mcp-voice-soundboard](https://github.com/mcp-tool-shop-org/mcp-voice-soundboard) भी कॉन्फ़िगर किया गया है, तो `voice_speak(mood: "roast")` इसे मूड के अनुसार उपयुक्त पाईपर लय के साथ बोलेगा।

## उपकरण

सभी उपकरण सत्र से वर्तमान मूड प्राप्त करते हैं।

| उपकरण | हस्ताक्षर | विवरण |
|------|-----------|-------------|
| `mood_set` | `(style: string)` | सक्रिय मूड सेट करें (सूखा, व्यंग्य, अराजक, शरारती, संशयवादी, ज़ूमर) |
| `mood_get` | `()` | वर्तमान मूड + चुटकुला गणना |
| `comic_timing` | `(text, technique?)` | हास्यपूर्ण तरीके से फिर से लिखें (तीन का नियम, दिशाभ्रम, वृद्धि, कॉलबैक, कम करके आंकना, ऑटो) |
| `roast` | `(target, context?)` | वर्तमान मूड की आवाज में स्नेही व्यंग्य, 1-5 की गंभीरता लौटाता है। संदर्भ: कोड, त्रुटि, विचार, स्थिति |
| `heckle` | `(target)` | संक्षिप्त, तीक्ष्ण ताना |
| `catchphrase_generate` | `(context?)` | पुन: प्रयोज्य अंश बनाएं (सत्र में संग्रहीत) |
| `catchphrase_callback` | `()` | सबसे अधिक उपयोग किए जाने वाले वाक्यांश का पुन: उपयोग करें (या शून्य) |
| `debug_status` | `()` | लाइव बैकएंड स्वास्थ्य (ओलामा पहुंच योग्य, मॉडल खींचा गया), हल किया गया कॉन्फ़िगरेशन, फ़ॉलबैक गणना और सत्र स्थिति |
| `session_reset` | `()` | सभी सत्र स्थितियों को रीसेट करें (मूड, चुटकुले, अंश, वाक्यांश, मोड़ काउंटर) |

**निम्न गुणवत्ता वाला आउटपुट (टाइप किया गया, मशीन द्वारा संसाधित करने योग्य):** जब कोई उपकरण वास्तविक मॉडल उत्पन्न नहीं कर पाता है, तो वह एक पूर्वनिर्धारित पंक्ति और `degraded: true` तथा एक `degraded_reason` लौटाता है, जो कि एक **सीमित सेट** से होता है। इस पर आधारित होकर, उपयोग करने वाला एजेंट विभिन्न विकल्पों का पता लगा सकता है: `safety-filter` (एक आपत्तिजनक/उपमा/छिपी हुई जानकारी को बदल दिया गया) · `connection` · `timeout` · `model-not-found` · `auth` · `rate-limit` · `server` · `http` · `json-parse` · `validation` · `exhausted` · `unknown`. एक वास्तविक पीढ़ी में **कोई** `degraded` ध्वज नहीं होता है - इसकी अनुपस्थिति सकारात्मक संकेत है। **सभी** हास्य उपकरण इसमें शामिल हैं, जिसमें `catchphrase_callback` (एक सुरक्षित विकल्प के साथ याद किया गया वाक्यांश चिह्नित किया जाता है, इसे कभी भी वास्तविक के रूप में प्रस्तुत नहीं किया जाता)। `roast`/`heckle` भी सक्रिय `मनोदशा` को दर्शाते हैं; `catchphrase_generate` `is_fresh` लौटाता है (`true` = नया, `false` = पहले से मौजूद सत्र का वाक्यांश पुन: उपयोग किया गया)।

एक बार स्वास्थ्य जांच के लिए `debug_status` कॉल करें: लाइव उपलब्धता (साथ ही `unreachable_reason` जब अनुपलब्ध हो - `connection` बनाम `auth` बनाम `timeout`), हल किया गया मॉडल/होस्ट/टाइमआउट, पीढ़ी के आंकड़े जिसमें दोनों `fallback_calls` (बैकएंड) **और** `safety_filter_fires` (कितनी बार सुरक्षा प्रणाली ने एक पंक्ति को बदल दिया) शामिल हैं, और एक `prompt_fingerprint` + `active_prompt_key` जो *सक्रिय* प्रॉम्प्ट टेक्स्ट + मॉडल से जुड़ते हैं ताकि आउटपुट में बदलाव का कारण प्रॉम्प्ट बनाम मॉडल परिवर्तन के लिए जिम्मेदार हो - और एक मौन प्रॉम्प्ट-संस्करण डाउनग्रेड (अनुरोधित v2 जो v1 पर वापस चला गया) दिखाई दे।

## मूड लय (पाईपर आवाज)

प्रत्येक मूड एक विशिष्ट पाईपर आवाज + लय कॉन्फ़िगरेशन पर मैप करता है:

| मूड | आवाज | लंबाई_स्केल | शोर_स्केल | शोर_डब्ल्यू_स्केल | वॉल्यूम | चरित्र |
|------|-------|-------------|-------------|---------------|--------|-----------|
| सूखा | en_GB-alan-medium | 1.15 | 0.3 | 0.3 | 0.9 | सपाट, थका हुआ, लयबद्ध |
| व्यंग्य | en_US-ryan-high | 0.95 | 0.667 | 0.8 | 1.0 | आत्मविश्वासी व्यंग्य |
| अराजक | en_US-lessac-high | 0.88 | 0.8 | 0.9 | 1.1 | समाचार एंकर निरर्थक बातें सुना रहा है |
| शरारती | en_GB-cori-high | 1.05 | 0.5 | 0.6 | 0.95 | गर्म, मजाकिया, चंचल इशारा |
| संशयवादी | en_GB-alan-medium | 1.25 | 0.2 | 0.2 | 0.8 | ठंडा, सपाट, शून्य आश्चर्य |
| ज़ूमर | en_US-lessac-high | 0.90 | 0.85 | 0.9 | 1.15 | तेज, जोर से, स्ट्रीमर ऊर्जा |

## पर्यावरण चर

```bash
# sensor-humor
SENSOR_HUMOR_DEBUG=true                # verbose prompt/response dumps
SENSOR_HUMOR_TIMEOUT_MS=30000          # Ollama call timeout in ms (default: 30000; invalid values fall back to default)
SENSOR_HUMOR_TEMPERATURE=0.55          # generation temperature, clamped 0.0-2.0 (default: 0.55)
SENSOR_HUMOR_PROMPT_VERSION=1          # prompt set version (only v1 ships today; other values fall back to v1)
SENSOR_HUMOR_MODEL=qwen2.5:7b         # Ollama model (default: qwen2.5:7b)
SENSOR_HUMOR_PERSIST=false             # persist session to ~/.sensor-humor/session.json (survives restart; 24h expiry)
SENSOR_HUMOR_SESSION_DIR=              # override the session directory (default: ~/.sensor-humor)
OLLAMA_HOST=http://127.0.0.1:11434    # Ollama API host (default: http://127.0.0.1:11434)
OLLAMA_API_KEY=                        # Bearer token for a remote/cloud Ollama (e.g. https://ollama.com); unset for local

# voice integration (in voice-soundboard)
VOICE_SOUNDBOARD_ENGINE=piper          # or kokoro (default)
VOICE_SOUNDBOARD_PIPER_MODEL_DIR=/path/to/piper/models
```

## अवलोकनीयता और डिबग

- प्रत्येक उपकरण कॉल संकेत भेजता है, कच्चा ओलामा प्रतिक्रिया, पार्स किया गया आउटपुट, सत्र अद्यतन लॉग करता है।
- आवाज: डिबग लॉग मूड के अनुसार लागू किए गए पाईपर पैरामीटर दिखाते हैं।
- `SENSOR_HUMOR_DEBUG=true` सेट करें ताकि सब कुछ दिखाई दे।

## गुणवत्ता नोट्स

- हास्य की गुणवत्ता कंकाल-आधारित प्रॉम्प्ट इंजीनियरिंग से आती है, न कि किसी एक मॉडल सेटिंग से - प्रत्येक मनोदशा एक पूर्वानुमान योग्य आकार को मजबूर करती है। अपने स्वयं के मॉडल/हार्डवेयर पर हिट दर मापें `scripts/ab-scorecard.ts` (SCORECARD.md में टेम्पलेट)।
- प्रॉम्प्ट-स्थिरता प्रतिगमन गेट (v1.2): v1 मनोदशा प्रॉम्प्ट **स्थिर** हैं ( `tests/scorecard-frozen-prompts.ts` द्वारा पिन किया गया - इसे बदलने के लिए, इसे `v2` पर अपडेट करें, कभी भी सीधे संपादित न करें)। एक नियतात्मक **रूप + सुरक्षा** सुनहरा सेट + आंकड़े `npm test` में चलते हैं (कोई बैकएंड नहीं); `npm run scorecard` लाइव सांख्यिकीय विचलन जांच चलाता है - प्रति-मनोदशा हिट-दर विल्सन अंतराल पर आधारित है जिसमें तीन मान होते हैं: PASS / FAIL / INCONCLUSIVE, और SPRT प्रारंभिक-रोक। यह संरचनात्मक अनुरूपता + सुरक्षा को मापता है, **न कि** हास्य (स्वचालित हास्य स्कोरिंग अविश्वसनीय है - सर्वश्रेष्ठ LLM बनाम मानव सहसंबंध ≈0.2)।
- उपमा/तुलना फ़िल्टर: पोस्ट-सत्यापन रेगुलर एक्सप्रेशन + पुन: प्रयास, फिर यदि कोई रिसाव बना रहता है तो मनोदशा-आधारित सुरक्षित विकल्प।
- कठोर भाषा फ़िल्टर: एक नियतात्मक शब्द-सूची रेगुलर एक्सप्रेशन **प्रत्येक** हास्य उपकरण पर एक *अंतिम गेट* के रूप में चलता है (कैचफ्रेज़ सहित), प्रत्येक पुन: प्रयास के बाद फिर से जांच की जाती है और किसी भी विकल्प को लागू करने से पहले लागू किया जाता है। पहचान पथ पहले डी-ऑब्फ़स्केट करता है - NFKC + शून्य-चौड़ाई/बिडी पट्टी + होमोग्लिफ़ फ़ोल्ड + लीट्सपीक फ़ोल्ड + इंट्रा-वर्ड-सेपरेटर पट्टी + संयोजन-चिह्न पट्टी - ताकि सामान्य बचाव (शून्य-चौड़ाई सम्मिलन, सिरिलिक/ग्रीक जैसे दिखने वाले अक्षर, पूर्ण चौड़ाई, `r3tard`, `re-tard`, `retárd`) शब्द सीमा को पार करके किसी आपत्तिजनक शब्द को न आने दें। यह एक नियतात्मक **आधार** है, सुरक्षा रेल नहीं - ईमानदार ऊपरी सीमा के लिए सुरक्षा और विश्वास देखें।
- नियतात्मक: JSON स्कीमा प्रवर्तन, खराब आउटपुट पर पुन: प्रयास, सभी उपकरणों में मनोदशा विरासत लागू की जाती है।
- आवाज: पाइपर मनोदशा के अनुसार उच्चारण पृथक्करण देता है (लंबाई/शोर/वॉल्यूम); कोकोरो विकल्प केवल गति प्रदान करता है।
- केवल डेवलपर उपकरण साइडकिक। हास्य व्यक्तिपरक है; यदि आवश्यक हो तो किसी भी मनोदशा को पर्यावरण चर के माध्यम से अक्षम करें या प्रॉम्प्ट को समायोजित करें।

## सुरक्षा और विश्वास

- **डिफ़ॉल्ट रूप से स्थानीय** - HTTP के माध्यम से `localhost` पर ओलामा से बात करता है। `OLLAMA_HOST` कहीं और इंगित कर सकता है (जैसे कि एक दूरस्थ/क्लाउड ओलामा); यह एकमात्र बाहरी आउटपुट है और ऑपरेटर की स्पष्ट पसंद है।
- **फ़ाइल सिस्टम** - डिफ़ॉल्ट रूप से कोई नहीं। `SENSOR_HUMOR_PERSIST=true` के साथ, यह एक फ़ाइल पढ़ता/लिखता है, `~/.sensor-humor/session.json` (निर्देशिका को `SENSOR_HUMOR_SESSION_DIR` से बदलें), जिसमें केवल आपके सत्र की हास्य स्थिति (बिट्स, चुटकुले, कैचफ्रेज़) शामिल हैं - कोई क्रेडेंशियल नहीं। फ़ाइल 24 घंटे के बाद स्वचालित रूप से समाप्त हो जाती है।
- **गुप्त जानकारी** - डिफ़ॉल्ट रूप से कोई नहीं। यदि आप `OLLAMA_HOST` को एक दूरस्थ/क्लाउड ओलामा पर इंगित करते हैं, तो `OLLAMA_API_KEY` सेट करें; यह पर्यावरण से पढ़ा जाता है और केवल उस होस्ट पर `Bearer` हेडर के रूप में भेजा जाता है - कभी भी लॉग नहीं किया जाता, संग्रहीत नहीं किया जाता या प्रतिध्वनित नहीं किया जाता ( `debug_status` केवल यह रिपोर्ट करता है कि एक कुंजी सेट है या नहीं, इसका मान नहीं)।
- **कोई टेलीमेट्री नहीं** - कुछ भी एकत्र या भेजा नहीं जाता है।
- **सत्र स्थिति डिफ़ॉल्ट रूप से मेमोरी में होती है** - जब सर्वर प्रक्रिया बंद हो जाती है तो समाप्त हो जाती है; `SENSOR_HUMOR_PERSIST` के साथ डिस्क पर बने रहने का विकल्प चुनें।
- **इनपुट सैनिटाइजेशन** - प्रॉम्प्ट इंजेक्शन से पहले सभी उपयोगकर्ता द्वारा प्रदान किए गए टेक्स्ट को सामान्य और सैनिटाइज किया जाता है: यूनिकोड NFKC फ़ोल्ड, शून्य-चौड़ाई/बिडी/फॉर्मेट वर्ण हटाए जाते हैं, सामान्य होमोग्लिफ़ को ASCII में बदल दिया जाता है, नई लाइनें हटाई जाती हैं, नियंत्रण वर्ण हटा दिए जाते हैं, लंबाई सीमित की जाती है।
- **आउटपुट फ़िल्टरिंग (नियतात्मक आधार + ईमानदार ऊपरी सीमा)** - एक बेस64-संग्रहीत शब्द-सूची रेगुलर एक्सप्रेशन प्रत्येक हास्य उपकरण पर एक अंतिम सुरक्षा गेट के रूप में चलता है (प्रत्येक पुन: प्रयास के बाद फिर से जांच की जाती है, किसी भी विकल्प को लागू करने से पहले लागू किया जाता है), और एक कॉलर-इनपुट विकल्प स्थिर, इनपुट-मुक्त पंक्ति में बदल जाता है बजाय प्रतिबंधित टोकन को प्रतिध्वनित करने के। पहचान पथ पहले डी-ऑब्फ़स्केट करता है, इसलिए सामान्य बचाव विफल हो जाते हैं: शून्य-चौड़ाई/बिडी सम्मिलन, होमोग्लिफ़ (सिरिलिक/ग्रीक/पूर्ण चौड़ाई), लीट्सपीक (`r3tard`), इंट्रा-वर्ड विभाजक (`re-tard`, `r.e.t.a.r.d`), और संयोजन डायक्रिटिक्स (`retárd`)। दूषित प्रविष्टियों को लोड होने पर छेड़छाड़ किए गए/विरासत वाले बने हुए सत्र से भी हटा दिया जाता है। **यह क्या नहीं करता:** यह एक नियतात्मक शब्द-सूची फ़िल्टर है, न कि एक सीखा हुआ वर्गीकरण - यह सूची के बाहर/नए आपत्तिजनक शब्दों, एकल-अक्षर रिक्ति (`r e t a r d`), ASCII-कला / स्थानिक अस्पष्टता, पूर्ण यूनिकोड-तुलनीय कवरेज या सिमेंटिक/जेलब्रेक-क्लास हमलों से बचाव नहीं करता है। इसे स्थानीय डेवलपर-हास्य उपकरण के लिए सर्वोत्तम प्रयास आधार के रूप में मानें, न कि अविश्वसनीय सार्वजनिक इनपुट के लिए मॉडरेशन गारंटी के रूप में।
- **उपकरण त्रुटि आकार** - रनटाइम/उपकरण त्रुटियां स्टूडियो संरचित त्रुटि आकार (`{code, message, hint, retryable}`) लौटाती हैं; ध्यान दें कि *इनपुट-स्कीमा* सत्यापन त्रुटियां (जैसे कि एक अमान्य `मनोदशा`) हैंडलर के चलने से पहले एमसीपी एसडीके द्वारा पकड़ी जाती हैं और एसडीके की मानक `InvalidParams` त्रुटि के रूप में सामने आती हैं, न कि इस आकार में।

## आर्किटेक्चर

```
Host LLM (Claude, etc.)
  | calls tool
  v
sensor-humor MCP server (TypeScript, stdio)
  | builds mood prompt + session state
  v
Ollama (qwen2.5:7b-instruct, local)
  | returns JSON (schema-enforced)
  v
sensor-humor validates -> updates session -> returns to host
  | host optionally calls
  v
mcp-voice-soundboard (Piper backend)
  | maps mood -> prosody preset
  v
Piper TTS (local ONNX) -> audio
```
